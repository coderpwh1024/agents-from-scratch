### 百炼：100 元总计能用多少？总计 1 亿 Token 要多少钱？

对话同时包含输入和输出，不能只用一个固定单价。为了能直接比较，下表统一采用常见的 **输入占 80%、输出占 20%** 的总 Token 结构，且按上表的**首档实时价格**计算，单位为人民币。

为便于建立直觉，表中“短问答”按 **2,000 总 Token / 次**（约 1,600 输入 + 400 输出）估算；“长文任务”按 **20,000 总 Token / 次**（例如读一篇较长资料后给出较完整的总结）估算。它们只是预算示例，不是固定消耗。

| 模型 ID | 100 元可用总 Token | 总计 1 亿 Token 费用 | 计算得到的综合单价（元 / 百万 Token） | 100 元大概能完成什么 |
| --- | ---: | ---: | ---: | --- |
| `qwen3.7-plus`（当前 8 折） | 约 3,906 万 | 256 元 | 2.56 | 约 1.95 万次短问答，或约 1,950 次长文任务 |
| `qwen3.6-plus` | 2,500 万 | 400 元 | 4.00 | 约 1.25 万次短问答，或约 1,250 次长文任务 |
| `qwen3.5-plus` | 6,250 万 | 160 元 | 1.60 | 约 3.13 万次短问答，或约 3,125 次长文任务 |
| `qwen-plus`（非思考） | 约 9,615 万 | 104 元 | 1.04 | 约 4.8 万次短问答，或约 4,800 次长文任务 |
| `qwen-plus`（思考） | 约 4,464 万 | 224 元 | 2.24 | 约 2.23 万次短问答，或约 2,230 次长文任务 |
| `qwen3.6-flash` | 约 4,167 万 | 240 元 | 2.40 | 约 2.08 万次短问答，或约 2,080 次长文任务 |
| `qwen3.5-flash` | 约 1.79 亿 | 56 元 | 0.56 | 约 8.93 万次短问答，或约 8,930 次长文任务 |
| `qwen-flash` | 约 2.38 亿 | 42 元 | 0.42 | 约 11.9 万次短问答，或约 1.19 万次长文任务 |

**总计 1 亿 Token 的直观规模**：约等于 **5 万次短问答**，或 **5,000 次长文任务**。同样的 1 亿 Token，选用不同模型时能完成的工作量相近，但总费用不同。

计算公式如下，Azure 的总 Token 表也使用同一公式，只是把人民币单价换成美元单价：

```text
综合单价（每百万总 Token）
= 输入单价 × 80% + 输出单价 × 20%

总计 1 亿 Token 费用 = 综合单价 × 100

100 元可用总 Token = 100 ÷ 综合单价 × 1,000,000
```

例如 `qwen-flash`：

```text
综合单价 = 0.15 × 80% + 1.5 × 20% = 0.42 元 / 百万总 Token
总计 1 亿 Token 费用 = 0.42 × 100 = 42 元
100 元可用总 Token = 100 ÷ 0.42 × 1,000,000 ≈ 2.38 亿 Token
```

如果你的业务输出很长（例如生成报告、文章、代码），输出占比可能达到 50%，实际费用会高于本表；如果主要是读取长资料、输出很短，则实际费用会更低。

> 这些数字只适用于首档上下文。单次输入超过模型所列的 128K 或 256K 阈值后，会进入更高价格档。Batch 调用适用时通常可将输入、输出成本减半。

### 旗舰补充

`qwen3.7-max` 的全球部署原价为输入 12、输出 36 元 / 百万 Token；当前官方页面标注限时 5 折，约为 **6 / 18 元 / 百万 Token**。其可用性取决于所选地域。

### LLM 计费与优惠

- LLM 按输入和输出 Token 分别计费；阶梯根据单次请求的输入 Token 数确定。
- `qwen3.7-plus`、`qwen-plus` 和 Flash 系列支持 Batch 调用时，输入、输出均按实时调用价的 50% 计费。
- 支持上下文缓存的模型，缓存命中输入可享折扣；该优惠不能与 Batch 优惠叠加。
- 上表主流模型在中国内地通常有 100 万 Token 免费额度，有效期为百炼开通后 90 天。

**1 亿向量 Token 的直观规模**：可向量化约 **20 万个检索片段**，或约 **1 万篇每篇 1 万 Token 的普通文档**。这主要是首次导入和增量更新知识库的成本，不是用户每次聊天都会重复产生的完整费用。

## 选型建议

- 日常客服、分类、信息抽取：`qwen-flash`
- 质量与成本平衡：`qwen-plus` 或 `qwen3.5-plus`
- 复杂推理任务：`qwen3.7-plus`
- 知识库 / RAG：`text-embedding-v4`，离线建库优先 Batch

### Azure 计费说明

- 上表为 Global 标准价格。Data Zone 一般比 Global 高约 10%，Regional 价格可能更高；部署时应在 Azure 定价页按实际区域确认。
- Batch API 的输入与输出费用通常为对应标准价的 50%，但以目标模型、部署类型实际支持情况为准。
- 缓存输入仅适用于支持 Prompt Caching 的模型和请求；不应把它作为全部输入的预算单价。
- Azure 也提供 Provisioned Throughput Units（PTU）按小时的容量定价；本表不包含该类保底容量费用。

## Azure 选型建议

- 复杂通用任务：`gpt-5`；要求成熟稳定、超长上下文时可选 `gpt-4.1`。
- 大多数生产业务：`gpt-5-mini` 或 `gpt-4.1-mini`。
- 分类、提取、批量轻任务：`gpt-5-nano`，并优先使用 Batch API。
- 图文理解：`gpt-4o`；成本敏感的图文理解：`gpt-4o-mini`。
- 新建知识库：优先 `text-embedding-3-small`；对召回质量要求更高时使用 `text-embedding-3-large`。
